# Instagram-Clone Application Context for AI Coding Assistants

This repository contains a full-stack, microservice-based clone of Instagram designed with modern, premium design aesthetics and robust performance guidelines.

---

## 🏛️ System Architecture

The application is split into multiple back-end microservices (built using Python/FastAPI/SQLAlchemy) and a single React single-page application (SPA) front-end (Vite/React/TailwindCSS/GSAP).

### 1. Front-end (`/insta-frontend`)
- **Core Stack**: React, Vite, Vanilla CSS + TailwindCSS, Lucide-React.
- **Animations**: GSAP (GreenSock Animation Platform) for high-end UI transitions, modal animations, and smooth entrances.
- **Routing**: `react-router-dom` utilizing path query routing (e.g., `?username=...` for visitor profile routing).
- **Asset Storage**: Media files are uploaded directly to MinIO buckets using pre-signed PUT URLs requested from the Post service.

### 2. Back-end Services
- **Authentication Service (`/Authentication-Server`)**: Handles sign-up, login, password hashes, and JWT generation.
- **User Service (`/User-Service`)**: Manages profile metadata (usernames, display names, bios, avatars), follow relationships (pending/accepted states), and privacy settings.
- **Post Service (`/Post-Server`)**: Manages feed compilation, likes, comments, nested comment replies, soft deletion of posts/comments, and generates pre-signed upload/download URLs for MinIO storage.
- **Chat Service (`/chat-server`)**: Handles real-time messaging, chat rooms, and message persistence.
- **Notification Service (`/Notification-Server`)**: Listens to Kafka topics and dispatches events like user followed, request accepted, comment liked, etc.

## ⚙️ Key Feature Implementation Details

### 1. Post Creation Flow (Images vs. Reels)
- Users can toggle between two upload modes in the **Create Post Modal**:
  - **Post (Images)**: Accept only image file types (`image/*`), allowing a selection of up to 10 images with a slider/carousel preview.
  - **Reel (Video)**: Accept only a single video file (`video/*`). Client-side validation checks if the video duration is ≤ 30 seconds before allowing progression.
- The media type is sent dynamically to the post endpoint as `media_type: "IMAGE"` or `media_type: "VIDEO"`.

### 2. Profile and Ownership Controls
- The **Profile Page** loads another user's posts via a custom `GET /posts/user/{user_id}` route using their retrieved profile UUID.
- Dynamic ownership checks compare the active post/comment `user_id` against the lightweight logged-in user profile (`me`) object.
- **Visitor Constraints**: When viewing someone else's profile, Edit and Delete controls for both posts and comments are automatically hidden or disabled unless the user is the author.

### 3. Media Rendering
- An `<AvatarImg>` helper component handles empty string fallbacks by displaying initials rather than triggering broken image network requests.
- Video files are displayed using native `<video>` elements with standard play controls.
